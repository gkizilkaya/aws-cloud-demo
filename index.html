01 — Temel Kavram

Bulut Bilişim Nedir?

Sunucu satın almak yerine, ihtiyaç duyduğunuzda kiralayıp kullandığınız kadar ödemek. AWS tam da bunu sağlar — fiziksel donanım sorumluluğu olmadan, ölçeklenebilir altyapı.

On-Premise (Eski Model)

Şirket kendi sunucularını satın alır, kurar, bakımını yapar. Pahalı, yavaş ölçeklenir. Donanım arızası veya afet durumunda veri kayıp riski yüksek.

Cloud (Yeni Model)

AWS'nin veri merkezlerindeki sunucular kiralanır. Saniyeler içinde yeni kaynak açılır, kullanılmadığında kapatılır. Yalnızca kullanılan kadar ödenir.

Neden Önemli?

Müşteri verisi nerede saklanacak, hangi sunucuda çalışacak, nasıl ölçeklenecek — bunlar bulut altyapısı kararları. Bu kararlar hem maliyeti hem güvenliği doğrudan etkiler.

02 — Ağ Güvenliği

VPC – Virtual Private Cloud

AWS içinde size özel, izole edilmiş bir ağ alanı. Dışarıdan gelen trafiğin nasıl yönleneceğini ve neyin içeri girebileceğini siz belirlersiniz.

Ne İşe Yarar?

Bir apartman gibi düşünün: AWS şehir, VPC apartman. Dışarıdan direkt odaya giremezsiniz. Hangi kapının açık olacağına siz karar verirsiniz.

Temel Bileşenler

  • CIDR Bloğu: VPC'nin IP aralığı. Örn: 10.0.0.0/16 = 65.536 adres
  • Subnet: VPC içindeki alt ağlar. Public = internete açık, Private = kapalı
  • Internet Gateway: VPC'nin internet kapısı
  • Route Table: Trafiğin nereye yönleneceğini belirler
  • Security Group: Instance bazlı güvenlik duvarı
  • NACL: Subnet bazlı ek güvenlik katmanı

Public vs Private Subnet: Web sunucusu public subnet'te olur — internetten istek alması gerekir. Veritabani private subnet'te olur — dışarıdan erişilmemeli, yalnızca web sunucusu erişmeli. Bu basit ayrım büyük güvenlik sağlar.
Dikkat — NAT Gateway: Private subnet'teki kaynakların internete çıkması için kullanılır. Saatlik ~$0.045 ücretlidir. Test ortamında seçmekten kaçinin.
03 — Sanal Sunucu

EC2 – Elastic Compute Cloud

AWS'nin sanal makine servisi. İstediğiniz boyutta, istediğiniz işletim sistemiyle dakikalar içinde sunucu açabilirsiniz.

Instance Tipleri

t3.micro — 2 vCPU, 1 GB RAM. Free Tier kapsamında. Basit uygulamalar için.

t3.medium — 2 vCPU, 4 GB RAM. Küçük üretim ortamları.

c6g.large — Hesaplama odaklı. ML çıkarım işlemleri için uygun.

AMI

Amazon Machine Image — sunucunuzun başlangıç şablonu. Hangi işletim sistemi, hangi yazılımlar yüklü. Amazon Linux 2023 AWS için optimize edilmiş, güncel ve ücretsiz.

Key Pair & SSH

Sunucuya bağlanmak için şifre yerine kriptografik anahtar (.pem dosyası) kullanılır. SSH ile uzak sunucuya terminal açılır. Anahtar dosyasını kaybetmemek kritiktir.

User Data

Instance ilk açildiginda otomatik çalışan script. Apache kurma, uygulama başlatma gibi işlemler burada tanımlanır. Bir kez yaz, her açılışta çalışsın.

Stop vs Terminate: Instance'ı "Durdur" (Stop) yaparsanız disk ücreti devam eder. Tamamen ücreti kesmek için "Sonlandır" (Terminate) seçin. Free Tier'da t3.micro için 750 saat/ay ücretsiz — sürekli açık 1 sunucu ~720 saat demektir, sınıra yakın.
04 — Serverless Container

AWS Fargate

Container çalıştırmak için sunucu yönetmek zorunda olmadığınız AWS servisi. Siz yalnızca uygulamayı tanımlarsınız, altyapıyı AWS yönetir.

Container nedir? Uygulamanızı ve tüm bağımlılıklarını (Python sürümü, kütüphaneler, konfigürasyon) bir pakete koyup her ortamda aynı şekilde çalıştırmayı sağlayan teknoloji. Docker ile oluşturulur.

EC2 mi, Fargate mi?

Kriter EC2 Fargate
Sunucu Yönetimi Siz yönetirsiniz — OS, güncelleme, patch AWS yönetir, sizin müdahaleniz gerekmez
Ölçekleme Auto Scaling Group manuel olarak kurulur Task bazlı, otomatik
Ücret Modeli Sunucu çalıştığı sürece (dursa bile disk ücreti) Yalnızca container çalıştığı süre
Başlangıç Süresi Dakikalar (instance boot) Saniyeler (container start)
GPU Desteği Evet — model eğitimi için gerekli Hayır
Kontrol Düzeyi Tam kontrol — SSH, özel sistem ayarları Sınırlı, ama çoğu kullanım için yeterli
Ne Zaman Tercih Edilir? GPU gereken işler, özel sistem konfigürasyonu Stateless API servisleri, CI/CD pipeline
