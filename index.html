02 · Ağ Güvenliği

VPC – Virtual Private Cloud

AWS içinde size özel, izole edilmiş bir ağ ortamı. İnternetten gelen her şey buradan geçer. Siz kural koyarsınız.

VPC Neye Benzer?

Bir apartman düşünün. AWS = şehir, VPC = apartman. Dışarıdan direkt odaya giremezsiniz. Kapıcı (Security Group), kat güvenliği (NACL) var. Siz kiracı olarak hangi kapıyı açacağınıza karar verirsiniz.

Temel Bileşenler

  • CIDR Bloğu: VPC'nin IP aralığı. Örn: 10.0.0.0/16 = 65.536 IP adresi
  • Subnet: VPC içindeki alt ağlar. Public = internete açık, Private = kapalı
  • Internet Gateway: VPC'nin internet kapısı
  • Route Table: Trafiğin nereye gideceğini belirler
  • Security Group: Instance bazlı güvenlik duvarı (hangi port açık?)
  • NACL: Subnet bazlı güvenlik duvarı (ek katman)

Public vs Private Subnet farkı: Web sunucusu (EC2) public subnet'te olur çünkü internet'ten istek alması gerekir. Veritabanı private subnet'te olur çünkü dışarıdan direkt erişilmemeli, sadece web sunucusu erişmeli. Bu basit kural büyük güvenlik sağlar.
⚠️ Dikkat: NAT Gateway (private subnet'teki kaynakların internete çıkması için) ücretlidir. Saatlik ~$0.045. Test ortamında "None" seçin.
03 · Sanal Sunucu

EC2 – Elastic Compute Cloud

AWS'nin sanal makine servisi. İstediğiniz boyutta (CPU, RAM), istediğiniz işletim sistemiyle, dakikalar içinde sunucu açabilirsiniz.

⚡

Instance Tipleri

t3.micro: 2 vCPU, 1GB RAM. Ücretsiz tier. Test için ideal.

t3.medium: 2 vCPU, 4GB RAM. Küçük prod için.

c6g.large: Hesaplama odaklı. ML inference için uygun.

🖼️

AMI (Makine Görüntüsü)

Sunucunuzun "fotoğrafı". Hangi işletim sistemi, hangi yazılımlar kurulu. Amazon Linux 2023 = AWS için optimize edilmiş, ücretsiz, güncel Linux.

🔑

Key Pair & SSH

Sunucuya bağlanmak için şifre yerine kriptografik anahtar kullanılır (.pem dosyası). SSH = Secure Shell, uzak sunucuya terminal açmak.

📝

User Data

Instance ilk başladığında otomatik çalışan script. Apache kurmak, uygulama deploy etmek gibi işlemler. Bir kez yaz, her açılışta çalışsın.

04 · Serverless Container

AWS Fargate

Container çalıştırmak için sunucu yönetmek zorunda olmadığınız AWS servisi. Siz sadece uygulamanızı tanımlarsınız, AWS gerisini halleder.

Container nedir? Uygulamanızı ve tüm bağımlılıklarını (Python, kütüphaneler, config) bir kutu içine koyup her yerde aynı şekilde çalıştırmanızı sağlayan teknoloji. Docker ile oluşturulur.

Fargate Nasıl Çalışır?

  • Task Definition: Container'ın ne kadar CPU/RAM kullanacağını, hangi Docker image'ı çalıştıracağını tanımlarsınız
  • Cluster: Task'ların çalıştığı mantıksal grup
  • Service: Task'ın kaç kopya çalışacağını, nasıl ölçekleneceğini yönetir
  • ECS: Elastic Container Service. Fargate'in arka planda kullandığı orkestrasyon katmanı

Fargate'in Avantajları

  • Sunucu yok: OS, patch, güvenlik güncellemesi derdi yok
  • Otomatik ölçekleme: Trafik artınca container sayısı artar
  • Sadece kullanım ücreti: Container çalıştığı süre kadar öde
  • Mikroservis uyumlu: Her servis ayrı container = kolay yönetim
  • CI/CD ile mükemmel: GitHub Actions → Docker build → Fargate deploy
